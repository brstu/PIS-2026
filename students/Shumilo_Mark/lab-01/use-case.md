# Use-case: Бронирование спортплощадки «Играем?» 🏀

**Первичный актор**: Пользователь (игрок, команда или организатор игры)

**Цель**: Забронировать свободную спортплощадку для проведения игры или тренировки.

**Предусловия**:
- Пользователь зарегистрирован в системе.
- В системе есть актуальное расписание доступных площадок.
- У пользователя есть доступ к интернету или мобильному приложению.

**Основной поток (Happy Path):**
1. Пользователь открывает приложение «Играем?».
2. Выбирает категорию площадки (например: баскетбольная, футбольная, теннисная).
3. Просматривает расписание доступных слотов.  
4. Выбирает удобное время и площадку.  
5. Нажимает кнопку «Забронировать».  
6. Система проверяет доступность выбранного слота.  
7. Система фиксирует бронь и отправляет подтверждение пользователю.  
8. Пользователь получает уведомление о бронировании (например, push-уведомление или email).  

**Постусловия:**  
- Выбранная площадка недоступна для других пользователей в указанное время.
- Пользователь имеет подтверждённую бронь.

**Альтернативные потоки:**
- 3a. Если пользователь хочет выбрать площадку по рейтингу/отзывам:
  - 3a1. Система отображает список площадок с рейтингами и отзывами.
  - 3a2. Пользователь выбирает площадку и возвращается к шагу 4.

- 5a. Если пользователь хочет пригласить друзей:
  - 5a1. Система предлагает добавить участников из списка контактов.
  - 5a2. После выбора участников система отправляет им приглашения.
  - 5a3. Возврат к шагу 6.

**Исключительные ситуации:**
- 6a. Если слот уже занят другим пользователем:
  - 6a1. Система уведомляет пользователя, что слот недоступен.
  - 6a2. Система предлагает выбрать другой слот.
  - 6a3. Use-case продолжается с шага 3.

- 7a. Если произошла ошибка при записи брони (например, сбой сервера):  
  - 7a1. Система откатывает транзакцию.  
  - 7a2. Система уведомляет пользователя об ошибке.  
  - 7a3. Use-case завершается неудачей.
