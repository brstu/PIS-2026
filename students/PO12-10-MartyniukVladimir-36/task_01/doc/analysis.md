# Анализ границ ответственности

## Таблица операций

| Операция | Синхр/Асинхр | Откат | Retry | Идемп. |
|----------|--------------|-------|-------|--------|
| Проверка авторизации | Синхр | Нет | Нет | Да |
| Проверка существования голоса | Синхр | Да | Нет | Да |
| INSERT в votes | Синхр | Да | Через БД | Да |
| UPDATE ideas | Синхр | Да | Через БД | Да |
| COMMIT | Синхр | Н/Д | 3 попытки | Да |
| Логирование | Асинхр | Нет | 2 попытки | Да |

## Точки отказа и их обработка

### 1. Duplicate key violation
- Условие: Пользователь отправил два запроса одновременно
- Реакция: ROLLBACK, возврат 409
- Уведомление: "Вы уже голосовали"

### 2. Deadlock
- Условие: Конкурентный доступ
- Реакция: Retry до 3 раз
- Уведомление: прозрачно для пользователя

### 3. Недоступность БД
- Условие: Потеря соединения
- Реакция: ROLLBACK, возврат 503
- Уведомление: "Сервис временно недоступен"

## Транзакционные границы

Транзакция начинается после проверок и включает:
1. INSERT INTO votes
2. UPDATE ideas SET likes_count = likes_count + 1

Эти операции должны быть атомарными - либо обе выполнены, либо ни одной.
