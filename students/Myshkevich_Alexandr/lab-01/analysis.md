5.2. Обработка исключительных ситуаций
Реализовано стратегий обработки: 7
Примеры:
Исключительная ситуация 1: Попытка отметить непоследовательный шаг
• Условие: Пользователь пытается отметить шаг №7, когда текущий шаг №5 ещё не выполнен
• Обнаружение: SQL запрос проверяет максимальный номер выполненного шага и сравнивает с запрашиваемым
• Реакция: Блокировка операции, показ подсказки
• Компенсация: Не требуется (данные не изменялись)
• Уведомление: "Сначала выполните текущий шаг: Основы CSS" ________________________________________
Исключительная ситуация 2: Сбой при сохранении прогресса (БД недоступна)
• Условие: PostgreSQL не отвечает на запросы (connection timeout)
• Обнаружение: Драйвер БД выбрасывает исключение при попытке UPDATE или COMMIT
• Реакция: ROLLBACK транзакции, сохранение данных в Redis
• Компенсация: Полный откат транзакции, асинхронное восстановление из Redis
• Уведомление: "⚠️ Прогресс временно не сохранился, но мы это запомнили. Обнови страницу через минуту" ________________________________________
Исключительная ситуация 3: Не удалось загрузить ресурсы для следующего шага
• Условие: Внешний сервис Resources Service не отвечает в течение 5 секунд
• Обнаружение: TimeoutException при вызове Resources Service
• Реакция: Логирование ошибки, пропуск загрузки материалов
• Компенсация: Не требуется (прогресс уже сохранён)
• Уведомление: "Шаг выполнен! Но материалы временно недоступны. Зайди позже." ________________________________________
Исключительная ситуация 4: Сервис достижений недоступен
• Условие: Achievement Service возвращает 500 или timeout
• Обнаружение: HTTP 500 или TimeoutException при вызове checkMilestones()
• Реакция: Логирование ошибки, создание фонового Job'а для повторной проверки
• Компенсация: Асинхронная проверка и выдача бейджа позже
• Уведомление: "Шаг выполнен! Прогресс: 25%" (без упоминания бейджа) ________________________________________
Исключительная ситуация 5: Не найдена конфигурация следующего шага
• Условие: В базе данных нарушена целостность (у шага №5 нет следующего шага №6)
• Обнаружение: SELECT следующего шага возвращает пустой результат
• Реакция: Логирование критической ошибки, завершение операции
• Компенсация: Не требуется (шаг уже отмечен)
• Уведомление: "Шаг выполнен, но следующий шаг не найден. Обратись в поддержку." ________________________________________
Исключительная ситуация 6: Конкурентный доступ (два пользователя отмечают один шаг)
• Условие: Два запроса одновременно пытаются отметить один и тот же шаг
• Обнаружение: Механизм блокировок на уровне БД (optimistic/pessimistic lock)
• Реакция: Второй запрос отклоняется с ошибкой
• Компенсация: Не требуется (первый запрос уже выполнен)
• Уведомление: "Этот шаг уже отмечен как выполненный" ________________________________________
•	Исключительная ситуация 7: Повторный запрос (идемпотентность)
• Условие: Пользователь дважды нажал кнопку из-за сетевой задержки
• Обнаружение: Проверка по idempotency_key или статусу шага
• Реакция: Возврат ранее сохранённого результата без изменений
• Компенсация: Не требуется
• Уведомление: "Шаг уже выполнен" пользователя
