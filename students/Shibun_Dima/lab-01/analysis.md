# Анализ границ ответственности

## Итоговая таблица

| Операция | Синхронная/Асинхронная | Откат при ошибке | Retry-стратегия | Идемпотентность |
|---------|-------------------------|------------------|------------------|------------------|
| Сохранение записи о воде в БД | Синхронная | Да (ROLLBACK) | Нет (контролируется БД) | Да (idempotency_key или user_id+timestamp) |
| Обновление дневного прогресса | Синхронная | Да (в той же транзакции) | Нет | Да |
| Отправка события в Analytics | Асинхронная | Нет (best-effort) | 5 попыток с exp. backoff | Да (дедупликация по event_id) |
| Отправка email-уведомления | Асинхронная | Нет | 5 попыток с exp. backoff | Да (по user_id+date) |
| Обновление долгосрочной статистики | Асинхронная | Нет | Повтор через очередь | Да |

# Часть 5. Обработка исключительных ситуаций

## Исключительная ситуация: Ошибка сохранения данных в БД

**Условие возникновения:**  
База данных недоступна или возвращает ошибку при попытке сохранить запись о потреблении воды.

**Обнаружение:**  
- API получает ошибку от БД (например, DBConnectionError или WriteFailure).  
- Логируется событие: "DB write error for user_id=<id>".

**Реакция:**  
1. Система откатывает транзакцию.  
2. Возвращает ошибку API уровня 500.  
3. Прогресс пользователя не обновляется.

**Компенсация:**  
- Нет частично выполненных действий, так как транзакция откатилась полностью.

**Уведомление пользователя:**  
"Не удалось сохранить данные. Попробуйте позже."


---

## Исключительная ситуация: Некорректный ввод объёма воды

**Условие возникновения:**  
Пользователь вводит отрицательное число, текст или слишком большой объём (например, 5000 мл).

**Обнаружение:**  
- Валидация на уровне UI или API.  
- API выбрасывает ValidationError.

**Реакция:**  
1. Система не выполняет запись в БД.  
2. Возвращает ошибку 400 (Bad Request).

**Компенсация:**  
- Не требуется, так как операция не начиналась.

**Уведомление пользователя:**  
"Введите корректный объём воды."


---

## Исключительная ситуация: Недоступность сервиса аналитики

**Условие возникновения:**  
Сервис Analytics не отвечает или возвращает ошибку.

**Обнаружение:**  
- HTTP‑клиент выбрасывает TimeoutException или 5xx.  
- Лог: "Analytics service unavailable".

**Реакция:**  
1. Основная транзакция НЕ откатывается.  
2. Событие ставится в очередь для повторной отправки.

**Компенсация:**  
- Повторная отправка через очередь (retry с экспоненциальной задержкой).

**Уведомление пользователя:**  
"Данные сохранены, но статистика обновится позже."


---

## Исключительная ситуация: Недоступность email‑сервиса

**Условие возникновения:**  
Email‑сервис не отвечает или возвращает ошибку.

**Обнаружение:**  
- HTTP‑клиент выбрасывает TimeoutException.  
- Лог: "Email service timeout".

**Реакция:**  
1. Основная транзакция НЕ откатывается.  
2. Уведомление ставится в очередь на повторную отправку.

**Компенсация:**  
- Повторная отправка через очередь (5 попыток с exponential backoff).

**Уведомление пользователя:**  
"Запись сохранена. Уведомление будет отправлено позже."
